\section{研究背景}

\begin{frame}{研究背景與核心問題}
\begin{itemize}
  \item 2021 年 1 月 6 日，美國國會山莊發生暴動，社群平台成為政治動員與情緒擴散的重要場域。
  \item 川普在事件前後的公開發言與推文，是否與 Twitter 上的討論量與情緒波動呈現明顯關聯，是本專題的核心問題。
  \item 本研究以 Python 建立完整分析流程，結合情緒分析、時間序列分群與視覺化圖表，觀察事件相關討論的情緒變化。
  \item 本專題目標不是宣稱單一貼文直接造成暴動，而是評估政治人物訊號是否具有推波助瀾的觀察證據。
\end{itemize}
\end{frame}

\begin{frame}{研究假設與分析主軸}
\begin{block}{研究假設}
當討論內容聚焦於川普、國會山莊、Pence 與 riot 等事件關鍵詞時，Twitter 討論將呈現較高的負向情緒比例，並在關鍵發文時點附近出現明顯放大量。
\end{block}

\vspace{0.6em}
\begin{itemize}
  \item 分析主體：2021-01-06 當日 Twitter 討論資料。
  \item 分析方法：VADER 情緒分析 + 15 分鐘時間窗聚合。
  \item 觀察重點：討論量、負向比例、正負情緒差距、關鍵詞分布。
  \item 論述方向：以資料支持「情緒確實受到事件與關鍵政治訊號牽動」。
\end{itemize}
\end{frame}

\begin{frame}{為何不能直接使用 Twitter API}
\begin{itemize}
  \item 直接依賴 Twitter 或 X Developer API 進行大量歷史資料補齊，會受到 endpoint rate limits 限制。
  \item 官方文件明確指出，每個 endpoint 都有獨立的請求上限，超出後會回傳 429，需等待視窗重設。
  \item 對於已刪除、受保護或遭停權內容，官方 API 的可近性也有限。
  \item 因此本專題不以線上逐筆查詢建立時間戳，而改採 TweetedAt 所採用的 Snowflake 逆向工程方式，由 tweet ID 還原時間。
\end{itemize}

\vspace{0.5em}
\tiny 來源：X API Rate Limits 文件；TweetedAt README。
\end{frame}

\begin{frame}{Snowflake 逆向工程的角色}
\begin{block}{核心概念}
Twitter Snowflake ID 是一種 64-bit、近似依時間排序的識別碼，包含 timestamp、worker number 與 sequence number。
\end{block}

\vspace{0.5em}
\begin{itemize}
  \item Twitter 於 2010 年公開說明 Snowflake 的設計，強調其 ID 具備 roughly sortable 的時間特性。
  \item TweetedAt 將這個機制應用於時間還原，從 tweet ID 反推出建立時間。
  \item 本專題沿用這個思路，在無法逐筆調用 API 的前提下，補齊 \texttt{created\_at} 欄位，才能進行時間序列分析。
  \item 因此，Snowflake 是本專題能夠成立的資料工程關鍵步驟。
\end{itemize}
\end{frame}

\begin{frame}{VADER 情緒分析設計}
\begin{itemize}
  \item VADER（Valence Aware Dictionary and sEntiment Reasoner）適合短文字、社群貼文與帶有情緒語氣的句子。
  \item 每則推文會得到一個 compound score，再依門檻分為 Positive、Neutral、Negative。
  \item 本專題優點在於流程穩定、可重跑、速度快，適合大量短文本做基礎情緒量化。
  \item 對政治諷刺、隱喻或雙關語仍有侷限，因此需要搭配時間脈絡與關鍵詞結果共同解讀。
\end{itemize}
\end{frame}

\section{程式碼講解}

\begin{frame}[fragile]{程式碼一：資料載入與去重}
\begin{CodeBlock}
def load_data(file_path=None):
    path = Path(file_path) if file_path else resolve_default_input_file()
    df = pd.read_csv(path)
    prepared = df.copy()
    prepared["text"] = prepared["text"].astype(str)
    prepared["created_at"] = pd.to_datetime(
        prepared["created_at"], errors="coerce", utc=True
    )
    prepared = prepared.dropna(subset=["created_at", "text"]).copy()
    prepared = prepared.drop_duplicates(subset=["tweet_id"])
    return prepared.sort_values("created_at").reset_index(drop=True)
\end{CodeBlock}

\begin{itemize}
  \item 先統一欄位格式，再刪除缺值與重複 tweet。
  \item 這一步確保時間排序與後續聚合結果一致。
\end{itemize}
\end{frame}

\begin{frame}[fragile]{程式碼二：事件子集合抽取}

def filter_event_subset(df, keywords=EVENT_KEYWORDS):
    pattern = "|".join(keywords)
    mask = df["text"].astype(str).str.contains(
        pattern, case=False, na=False, regex=True
    )
    subset = df.loc[mask].copy()
    return subset.sort_values("created_at").reset_index(drop=True)
\end{CodeBlock}

\begin{itemize}
  \item 以事件關鍵詞建立與國會山莊事件高度相關的討論樣本。

\begin{frame}{結果解讀}
\begin{block}{主要發現}
在 2021 年 1 月 6 日的事件相關子集合中，Twitter 討論呈現更高比例的負向情緒，且在川普關鍵訊號與事件升高時段附近出現顯著討論量峰值。
\end{block}

\vspace{0.6em}
\begin{itemize}
  \item 事件子集合平均情緒分數為 -0.1423，顯示整體情緒偏負面。
  \item 討論量峰值與事件升溫時間接近，說明政治人物訊號與社群反應具有同步放大關係。
  \item 因為負向比例大幅高於整體資料基準，可合理推論事件相關內容確實對社群情緒造成衝擊。
\end{itemize}
\end{frame}

\begin{frame}{研究限制與方法邊界}
\begin{itemize}
  \item 本資料集不是川普官方帳號推文全量資料，因此不宜宣稱嚴格因果。
  \item 事件子集合使用關鍵詞篩選，仍可能混入評論、轉述與媒體報導內容。
  \item VADER 對政治諷刺與上下文依賴語句的判斷仍有誤差。
  \item 因此本研究最適合的表述是：川普相關政治訊號與社群情緒升高呈現明顯關聯，具有推波助瀾的觀察證據。
\end{itemize}
\end{frame}

\begin{frame}{結論}
\begin{enumerate}
  \item 本專題成功以 Python 建立可重跑的 Twitter 情緒分析流程，完成資料整理、Snowflake 時間還原、VADER 評分與圖表輸出。
  \item 在 1 月 6 日國會山莊事件相關子集合中，負向情緒比例由整體資料的 33.34\% 上升到 54.12\%。
  \item 事件相關討論在關鍵發文時段附近出現大規模放量，支持「政治人物訊號會強化社群情緒波動」的研究判斷。
  \item 綜合本次資料，本專題結論傾向認為：川普相關發文與國會山莊事件討論之間，存在明顯的情緒放大效果。
\end{enumerate}
\end{frame}
